# Отчет – Лабораторной работы Nr.7

---

**Тема работы:** Создание контейнеров. Виртуализация в Docker.

---

**Цель работы** – формирование практических навыков использования технологий
виртуализации на уровне контейнеров с помощью платформы Docker для создания,
настройки и администрирования контейнерных приложений в контролируемой и
воспроизводимой среде.

- Понимание концептуальных различий между традиционной
  виртуализацией и виртуализацией на основе контейнеров, а также развитие навыков
  автоматизации и оркестрации среды выполнения.

---

**Цели работы**

- Объяснение концепции контейнеризации и ее сравнение с классической
  виртуализацией на основе гипервизоров.
- Установка и настройка платформы Docker на операционной системе
  (Linux/Windows/Mac).
- Создание и управление контейнерами Docker с помощью командной строки
  (docker run, docker ps, docker stop, docker rm и т. д.).
- Создание собственных образов Docker с помощью файла Dockerfile.
- Управление хранилищами образов (Docker Hub, локальный реестр) —
  загрузка, выгрузка и версионирование.
- Создание сетей и томов Docker для связи между контейнерами и сохранения
  данных.
- Анализ производительности и изоляции контейнеров по сравнению с
  традиционными виртуальными машинами.

---

## Основная задача:

Основная задача этой лабораторной работы — создать как минимум один образ
и как минимум один контейнер с использованием технологии Docker в качестве
элемента виртуализации, а также решить задачу с точки зрения сложности.
Конечный результат должен содержать файл docker, а также программу для
решения выбранного варианта.

---

### Варианты реализации:

- **Легкий вариант:** Создайте одностраничный веб-сайт (CV), реализуйте
  проект в контейнере Docker и обеспечьте доступ к браузеру. (Nginx или Apache)
- **Средний:** Создайте веб-сайт, как указано в простом варианте, плюс собственное
  изображение в другом контейнере. Предоставьте доступ через браузер или консоль и
  продемонстрируйте функциональность.
- **Сложный вариант:** Создайте веб-сайт, как указано в простом варианте, плюс
  свои изображения во втором контейнере и CSS (каскадные таблицы стилей) в третьем
  контейнере. Предоставьте доступ через браузер или консоль и продемонстрируйте
  функциональность.

  Для всех вариантов выполните задание лабораторной работы 1 в команде в
  контейнере Docker.

---

## Критерии оценки:

1. Создание схемы в соответствии с соответствующим вариантом.
2. Создание контейнера Docker из легкого варианта .
3. Создание контейнера Docker из средней опции.
4. Создание контейнера Docker из легкой опции.
5. Корректность контейнера Docker — проверка того, что контейнер работает
   корректно, без ошибок.
6. Оптимизация контейнера Docker — оценка эффективности контейнера в
   использовании ресурсов.
7. Соблюдение срока сдачи — оценка баллов по пунктуальности, если работа была
   сдана в установленный срок.
8. Оценка знаний — объяснения, данные о процессе выполнения работы, что
   может включать описание основных функций и использованной логики.
9. Использование студентом ИИ.

_Для получения оценки 5 - 6_ обязательны критерии 1,2,5,7,8.
_Для получения оценки 7-8_ обязательны критерии 1, 3, 5, 7, 8.
_Для получения оценки 9-10_ обязательны критерии 1-8.

- Если был использован критерий 10, оценка снижается на 2 балла, только если студент
  разъяснил работу кода. В противном случае лабораторная работа не принимается.

- После завершения работы напишите отчет, который должен включать: имя, фамилию,
  группу, тему работы, вариант реализации задачи, краткое описание реализации задачи,
  ссылку на исходный код на GitHub. Запишите исходный код в свою ветку в
  соответствующем репозитории на GitHub. Сохраните отчет в формате PDF и отправьте
  его в ELSE.
