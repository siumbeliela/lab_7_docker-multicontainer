# ВВЕДЕНИЕ

Данный отчет составлен по результатам выполнения лабораторной работы №7 на тему **«Создание контейнеров. Виртуализация в Docker»**.

Цель работы — сформировать практические навыки использования технологий контейнеризации с помощью Docker, научиться создавать собственные образы, запускать контейнеры и управлять многоконтейнерными приложениями через Docker Compose.

В рамках лабораторной работы был реализован **сложный вариант задания** — создание многоуровневого веб-приложения, где каждый ресурс (HTML, CSS, изображения) обслуживается отдельным контейнером Nginx.

Используемые сервисы:

- **web-app** — HTML-сервис, порт 8080
- **css-server** — сервер стилей CSS, порт 8081
- **images-server** — сервер изображений, порт 8082

Контейнеры изолированы и взаимодействуют между собой через виртуальную сеть Docker.

## 1.1 Архитектура и структура приложения

В этом подразделе будет представлен обзор многокомпонентной архитектуры, где каждый статический ресурс (HTML-файл, файл стилей CSS и изображение) обслуживается отдельным, полностью изолированным контейнером Nginx. Также будет показана иерархия файлов и каталогов проекта (листинг файлов), которая необходима для корректной работы Docker Compose и сборки индивидуальных образов.
Проект реализован в следующей структуре каталогов и файлов:

### Цель архитектуры

Создать распределённое приложение, состоящее из нескольких сервисов, которые взаимодействуют друг с другом через Docker Network.

### Проект реализован в следующей структуре каталогов и файлов:

```yaml
lab/
│── html/
│ ├── index.html
│ ├── Dockerfile
│
│
│── images/
│ ├── my_image.jpg
│ └── Dockerfile
│
│── css/
│ ├── style.css
│ └── Dockerfile
│
└── docker-compose.yml
```

---

**Проект использует docker-compose.yml и разделен на три подкаталога для изоляции ресурсов:**

- **css/:** - Содержит Dockerfile и style.css.

`css-server (CSS-сервис): Обслуживает файл style.css через отдельный контейнер Nginx. Доступен на порту 8081.`

- **html/:** - Содержит Dockerfile и index.html.

`web-app (HTML-сервис): Основной веб-сайт (CV), использующий базовый образ nginx:alpine. Обслуживает только файл index.html и доступен на порту 8080.`

- **images/:** - Содержит Dockerfile и my_image.jpg

`images-server (Сервер изображений): Обслуживает файл my_image.jpg через третий контейнер Nginx. Доступен на порту 8082.`

## 1.2 Схема реализации

Схема иллюстрирует взаимодействие между компонентами: хост-системой, виртуальной сетью Docker и тремя изолированными контейнерами (web-app, css-server, images-server). Особое внимание уделяется пробросу портов (8080, 8081, 8082), который обеспечивает доступ браузера хост-машины к каждому сервису.

**Хост-система:** - Запускает Docker Engine и Docker Compose.
**Docker Network (по умолчанию):** - Все три контейнера находятся в одной виртуальной сети, что позволяет им обращаться друг к другу по именам сервисов `(css-server, images-server).`
**Контейнеры:** - web-app, css-server и images-server с корректными пробросами портов (`8080, 8081, 8082`), что подтверждает работоспособность конфигурации `docker-compose.yml.`

- `web-app (8080)`: Обслуживает index.html. Зависит от других.
- `css-server (8081)`: Обслуживает style.css.
- `images-server (8082)`: Обслуживает my_image.jpg.
- **Взаимодействие:** - Браузер на хост-машине запрашивает:
- Основную страницу: http://localhost:8080 (обращается к web-app).
- CSS-ресурс: http://localhost:8081/style.css (обращается к css-server).
- Графический ресурс: http://localhost:8082/images/my_image.jpg (обращается к images-server).

Основным инструментом объединения всех служб является файл `docker-compose.yml`.

### Cхема `docker-compose.yml`:

```yaml
version: "3.8"

services:
  web:
    build: ./web
    ports:
      - "8080:80"
    depends_on:
      - backend
    networks:
      - appnet

  backend:
    build: ./backend
    ports:
      - "5000:5000"
    depends_on:
      - image
    networks:
      - appnet

  image:
    build: ./image
    ports:
      - "9000:9000"
    networks:
      - appnet

networks:
  appnet:
    driver: bridge
```

## 1.3 Демонстрация запущенных контейнеров в приложении Docker Desktop

**После запуска командой:**
`docker-compose up --build `

**Так же в Docker Desktop можно наблюдать** три контейнера, работающие одновременно:

- `web-app (8080)`
- `css-server (8081)`
- `images-server (8082)`

---

**У каждого контейнера должны отображаться:**

- открытые порты (например, `8080, 5000, 9000`)
- собственные логи работы
- графики потребления RAM и CPU
- подключённые сети тома (если используются)

  ***

**Взаимодействие можно протестировать:**

- открыть браузер → http://localhost:8080
- нажать кнопку, которая делает запрос на backend
- результат (изображение или данные) должен быть показан пользователю

`Если все контейнеры отображаются зелёными — система работает корректно.`

Работа подтвердила понимание основных концепций Docker, включая жизненный цикл образа и контейнера, а также преимущества использования контейнеров для создания легковесных, переносимых и изолированных сред разработки и развертывания.
